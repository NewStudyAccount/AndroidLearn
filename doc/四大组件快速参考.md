# Android 四大组件快速参考手册

## 一、Activity（活动）

### 核心概念
Activity 是用户看到的每一个屏幕，是 Android 应用的核心组件。

### 生命周期方法

| 方法 | 调用时机 | 典型用途 |
|------|----------|----------|
| `onCreate()` | Activity 首次创建 | 初始化 UI、绑定数据 |
| `onStart()` | Activity 变为可见 | 注册广播接收器 |
| `onResume()` | Activity 进入前台可交互 | 恢复暂停的动画/视频 |
| `onPause()` | Activity 失去焦点 | 暂停动画、保存草稿 |
| `onStop()` | Activity 完全不可见 | 释放资源、停止位置更新 |
| `onDestroy()` | Activity 销毁前 | 清理所有资源 |
| `onRestart()` | 从停止状态重启 | 恢复暂停的操作 |

### 页面跳转

```kotlin
// 简单跳转
val intent = Intent(this, SecondActivity::class.java)
startActivity(intent)

// 带数据跳转
val intent = Intent(this, SecondActivity::class.java)
intent.putExtra("key", "value")
startActivity(intent)

// 获取返回结果
val launcher = registerForActivityResult(
    ActivityResultContracts.StartActivityForResult()
) { result ->
    if (result.resultCode == RESULT_OK) {
        val data = result.data?.getStringExtra("key")
    }
}
launcher.launch(intent)
```

### 启动模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `standard` | 默认，每次创建新实例 | 普通页面 |
| `singleTop` | 栈顶复用 | 搜索页、浏览器 |
| `singleTask` | 栈内复用 | 主页 |
| `singleInstance` | 独占任务栈 | 来电、闹钟 |

## 二、Service（服务）

### 核心概念
Service 用于执行后台任务，没有用户界面。

### 两种启动方式

| 方式 | 特点 | 生命周期 |
|------|------|----------|
| `startService()` | 独立运行，需手动停止 | 调用者销毁不影响 |
| `bindService()` | 与调用者绑定 | 调用者销毁则停止 |

### startService 示例

```kotlin
// 启动服务
val intent = Intent(this, MyService::class.java)
startService(intent)

// 停止服务
stopService(intent)
```

### bindService 示例

```kotlin
// 绑定服务
val intent = Intent(this, MyBindService::class.java)
bindService(intent, serviceConnection, Context.BIND_AUTO_CREATE)

// 解绑服务
unbindService(serviceConnection)
```

### 服务生命周期

```
startService: onCreate() → onStartCommand() → onDestroy()
bindService:  onCreate() → onBind() → onUnbind() → onDestroy()
```

## 三、BroadcastReceiver（广播接收器）

### 核心概念
用于接收系统或应用发出的广播消息。

### 常见系统广播

| 广播 | Action | 说明 |
|------|--------|------|
| 开机完成 | `ACTION_BOOT_COMPLETED` | 系统启动完成 |
| 电量变化 | `ACTION_BATTERY_LOW` | 电量低 |
| 网络变化 | `CONNECTIVITY_ACTION` | 网络状态改变 |
| 飞行模式 | `ACTION_AIRPLANE_MODE_CHANGED` | 飞行模式开关 |

### 两种注册方式

| 方式 | 特点 | 生命周期 |
|------|------|----------|
| 静态注册 | AndroidManifest.xml | 应用未启动也能接收 |
| 动态注册 | 代码中注册 | 跟随组件生命周期 |

### 静态注册

```xml
<receiver android:name=".MyReceiver" android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED" />
    </intent-filter>
</receiver>
```

### 动态注册

```kotlin
val receiver = MyReceiver()
val filter = IntentFilter(Intent.ACTION_BATTERY_LOW)
registerReceiver(receiver, filter)

// 取消注册
unregisterReceiver(receiver)
```

### 发送自定义广播

```kotlin
val intent = Intent("com.example.CUSTOM_ACTION")
intent.putExtra("message", "Hello")
intent.setPackage(packageName)  // Android 8.0+ 必须
sendBroadcast(intent)
```

## 四、ContentProvider（内容提供器）

### 核心概念
用于在不同应用之间共享数据。

### 常见系统 ContentProvider

| Provider | URI | 说明 |
|----------|-----|------|
| 通讯录 | `ContactsContract.Contacts.CONTENT_URI` | 联系人数据 |
| 媒体库 | `MediaStore.Images.Media.EXTERNAL_CONTENT_URI` | 图片 |
| 短信 | `Telephony.Sms.CONTENT_URI` | 短信数据 |

### 查询数据

```kotlin
val cursor = contentResolver.query(
    uri,           // 内容 URI
    projection,    // 要查询的列
    selection,     // WHERE 条件
    selectionArgs, // WHERE 参数
    sortOrder      // 排序
)

cursor?.use {
    while (it.moveToNext()) {
        val name = it.getString(it.getColumnIndex("name"))
    }
}
```

### 自定义 ContentProvider

```kotlin
class MyProvider : ContentProvider() {
    override fun onCreate(): Boolean { /* 初始化 */ }
    override fun query(...): Cursor? { /* 查询 */ }
    override fun insert(...): Uri? { /* 插入 */ }
    override fun update(...): Int { /* 更新 */ }
    override fun delete(...): Int { /* 删除 */ }
    override fun getType(...): String { /* MIME类型 */ }
}
```

## 学习建议

1. **先学 Activity** - 最重要，90% 的开发都在和 Activity 打交道
2. **Service 了解即可** - 现在很多后台任务用 WorkManager 替代
3. **BroadcastReceiver 了解即可** - 静态注册受限，动态注册更灵活
4. **ContentProvider 了解即可** - 除非需要跨应用共享数据
