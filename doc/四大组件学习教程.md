## 一、Activity 详解（最重要）

### 1.1 Activity 生命周期

```
                   ┌──────────────────────────────────────────┐
                   │              Activity 生命周期             │
                   └──────────────────────────────────────────┘

        ┌─────────────────────────────────────────────────────────────┐
        │                                                             │
        │   onCreate()  ──→  onStart()  ──→  onResume()              │
        │       ↑               ↑              ↑                      │
        │    首次创建         变为可见        获取焦点                   │
        │   (初始化UI)     (用户还不能交互)   (可以交互了)              │
        │                                                             │
        │                      ┌─────────┐                           │
        │                      │  运行中  │                           │
        │                      │ 用户交互 │                           │
        │                      └────┬────┘                           │
        │                           │                                │
        │                      onPause()  ←── 失去焦点               │
        │                           │          (如弹出对话框)         │
        │                      onStop()   ←── 不可见                 │
        │                           │          (如跳转到其他页面)      │
        │                      onDestroy() ←── 销毁                  │
        │                                    (Activity 被销毁)        │
        │                                                             │
        └─────────────────────────────────────────────────────────────┘

生命周期回调方法说明：
┌──────────────┬─────────────────────────────────────────────────────┐
│    方法      │                    说明                              │
├──────────────┼─────────────────────────────────────────────────────┤
│ onCreate()   │ Activity 首次创建时调用，用于初始化（加载布局、绑定数据）│
│ onStart()    │ Activity 变为可见时调用，但还不能与用户交互            │
│ onResume()   │ Activity 进入前台，可以与用户交互                     │
│ onPause()    │ Activity 失去焦点时调用（如弹出对话框）                │
│ onStop()     │ Activity 完全不可见时调用                            │
│ onDestroy()  │ Activity 被销毁前调用，释放资源                       │
│ onRestart()  │ Activity 从停止状态重新启动时调用                     │
└──────────────┴─────────────────────────────────────────────────────┘
```

### 1.3 页面跳转与数据传递

#### 方式一：使用 Intent 传递简单数据

```kotlin
// MainActivity.kt - 发送数据
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        val button = findViewById<Button>(R.id.btnGoToSecond)
        button.setOnClickListener {
            // 创建 Intent
            val intent = Intent(this, SecondActivity::class.java)

            // 传递数据
            intent.putExtra("name", "小明")
            intent.putExtra("age", 18)
            intent.putExtra("score", 95.5)

            // 启动 Activity
            startActivity(intent)
        }
    }
}

// SecondActivity.kt - 接收数据
class SecondActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_second)

        // 接收数据
        val name = intent.getStringExtra("name") ?: ""
        val age = intent.getIntExtra("age", 0)
        val score = intent.getDoubleExtra("score", 0.0)

        // 显示数据
        val textView = findViewById<TextView>(R.id.tvInfo)
        textView.text = "姓名: $name\n年龄: $age\n分数: $score"
    }
}
```

#### 方式二：使用 Bundle 传递数据

```kotlin
// 发送数据
```

#### 方式三：传递对象（Parcelable）

```kotlin
// MainActivity.kt - 启动 Activity 并等待结果
class MainActivity : AppCompatActivity() {

    private val startForResult = registerForActivityResult(

        if (result.resultCode == RESULT_OK) {

        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        findViewById<Button>(R.id.btnPick).setOnClickListener {
            val intent = Intent(this, PickActivity::class.java)
            startForResult.launch(intent)
        }
    }
}
```

### 1.5 Activity 启动模式

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Activity 启动模式                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. standard（默认）                                                 │
│     • 每次启动都创建新的 Activity 实例                                │
│     • 任务栈中可以有多个相同 Activity                                 │
│                                                                     │
│  2. singleTop                                                       │
│     • 如果 Activity 已在栈顶，不会创建新实例，调用 onNewIntent()       │
│     • 如果不在栈顶，会创建新实例                                      │
│     • 适用场景：浏览器书签、搜索页                                    │
│                                                                     │
│  3. singleTask                                                       │
│     • 任务栈中只存在一个 Activity 实例                                │
│     • 如果已存在，会清除它上面的所有 Activity                         │
│     • 适用场景：主页（MainActivity）                                  │
│                                                                     │
│  4. singleInstance                                                   │
│     • Activity 独占一个任务栈                                         │
│     • 适用场景：来电界面、闹钟                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**在 AndroidManifest.xml 中配置启动模式：**

```xml
<activity
    android:name=".SingleTaskActivity"
    android:launchMode="singleTask" />

<activity
    android:name=".SingleTopActivity"
    android:launchMode="singleTop" />
```

## 二、Service 详解

### 2.1 Service 概述

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Service 简介                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Service 是 Android 的后台服务组件，用于执行长时间运行的操作           │
│                                                                     │
│  特点：                                                             │
│  • 没有用户界面                                                     │
│  • 在后台运行（播放音乐、下载文件、定时任务等）                       │
│  • 即使用户切换到其他应用，Service 也能继续运行                       │
│                                                                     │
│  两种启动方式：                                                     │
│  ┌──────────────────┬─────────────────────────────────────────┐    │
│  │    startService  │  启动后独立运行，需要手动停止              │    │
│  │    bindService   │  与调用者绑定，调用者退出后自动停止        │    │
│  └──────────────────┴─────────────────────────────────────────┘    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 启动方式一：startService

```kotlin
// MyService.kt - 自定义服务
class MyService : Service() {

    private val TAG = "MyService"

    // 必须实现，返回 null 表示不支持绑定
    override fun onBind(intent: Intent?): IBinder? {
        return null
    }

    // 服务创建时调用（只调用一次）
    override fun onCreate() {
        super.onCreate()
        Log.d(TAG, "onCreate: 服务创建")
    }

    // 每次启动服务时调用
    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        Log.d(TAG, "onStartCommand: 服务启动")

        // 在这里执行后台任务
        Thread {
            for (i in 1..10) {
                Log.d(TAG, "后台任务执行中: $i")
                Thread.sleep(1000)
            }
            // 任务完成后停止服务
            stopSelf()
        }.start()

        // 返回值决定服务被杀死后的重启策略
        return START_STICKY  // 服务被杀死后会自动重启
    }

    // 服务销毁时调用
    override fun onDestroy() {
        super.onDestroy()
        Log.d(TAG, "onDestroy: 服务销毁")
    }
}
```

```kotlin
// 在 Activity 中启动/停止服务
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // 启动服务
        findViewById<Button>(R.id.btnStartService).setOnClickListener {
            val intent = Intent(this, MyService::class.java)
            startService(intent)
        }

        // 停止服务
        findViewById<Button>(R.id.btnStopService).setOnClickListener {
            val intent = Intent(this, MyService::class.java)
            stopService(intent)
        }
    }
}
```

```xml
<!-- AndroidManifest.xml 中注册 Service -->
<service android:name=".MyService" />
```

### 2.3 启动方式二：bindService

```kotlin
// MyBindService.kt
class MyBindService : Service() {

    // 创建 Binder 对象，用于与 Activity 通信
    private val binder = LocalBinder()

    inner class LocalBinder : Binder() {
        fun getService(): MyBindService = this@MyBindService
    }

    override fun onBind(intent: Intent?): IBinder {
        Log.d("MyBindService", "onBind: 服务绑定")
        return binder
    }

    override fun onUnbind(intent: Intent?): Boolean {
        Log.d("MyBindService", "onUnbind: 服务解绑")
        return super.onUnbind(intent)
    }

    // 服务中的方法，可以被 Activity 调用
    fun getData(): String {
        return "来自服务的数据"
    }
}
```

```kotlin
// Activity 中绑定服务
class BindActivity : AppCompatActivity() {

    private var myService: MyBindService? = null
    private var isBound = false

    // 服务连接回调
    private val connection = object : ServiceConnection {
        override fun onServiceConnected(name: ComponentName?, service: IBinder?) {
            val binder = service as MyBindService.LocalBinder
            myService = binder.getService()
            isBound = true
            Log.d("BindActivity", "服务已绑定")
        }

        override fun onServiceDisconnected(name: ComponentName?) {
            myService = null
            isBound = false
            Log.d("BindActivity", "服务已断开")
        }
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_bind)

        // 绑定服务
        findViewById<Button>(R.id.btnBind).setOnClickListener {
            val intent = Intent(this, MyBindService::class.java)
            bindService(intent, connection, Context.BIND_AUTO_CREATE)
        }

        // 调用服务方法
        findViewById<Button>(R.id.btnGetData).setOnClickListener {
            if (isBound) {
                val data = myService?.getData()
                Toast.makeText(this, data, Toast.LENGTH_SHORT).show()
            }
        }

        // 解绑服务
        findViewById<Button>(R.id.btnUnbind).setOnClickListener {
            if (isBound) {
                unbindService(connection)
                isBound = false
            }
        }
    }

    override fun onDestroy() {
        super.onDestroy()
        if (isBound) {
            unbindService(connection)
            isBound = false
        }
    }
}
```

### 2.4 前台服务（显示通知）

```kotlin
class MusicService : Service() {

    override fun onCreate() {
        super.onCreate()
        startForeground()
    }

    private fun startForeground() {
        // 创建通知渠道（Android 8.0+）
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            val channel = NotificationChannel(
                "music_channel",
                "音乐播放",
                NotificationManager.IMPORTANCE_LOW
            )
            val manager = getSystemService(NotificationManager::class.java)
            manager.createNotificationChannel(channel)
        }

        // 创建通知
        val notification = NotificationCompat.Builder(this, "music_channel")
            .setContentTitle("正在播放音乐")
            .setContentText("歌曲名称")
            .setSmallIcon(R.drawable.ic_music)
            .build()

        // 启动前台服务
        startForeground(1, notification)
    }

    override fun onBind(intent: Intent?): IBinder? = null
}
```

## 三、BroadcastReceiver 详解

### 3.1 广播接收器概述

```
┌─────────────────────────────────────────────────────────────────────┐
│                    BroadcastReceiver 简介                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  广播接收器用于接收系统或应用发出的广播消息                           │
│                                                                     │
│  常见系统广播：                                                     │
│  ┌────────────────────────────────────────────────────────────┐    │
│  │ • 电量变化 ACTION_BATTERY_CHANGED                          │    │
│  │ • 网络变化 CONNECTIVITY_ACTION                             │    │
│  │ • 开机完成 ACTION_BOOT_COMPLETED                          │    │
│  │ • 短信到来 SMS_RECEIVED                                    │    │

│  └────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  两种注册方式：                                                     │
│  ┌──────────────────┬─────────────────────────────────────────┐    │
│  │   静态注册       │  在 AndroidManifest.xml 中注册            │    │
│  │                  │  应用未启动也能接收广播                   │    │
│  ├──────────────────┼─────────────────────────────────────────┤    │
│  │   动态注册       │  在代码中注册                            │    │
│  │                  │  跟随组件生命周期，更灵活                 │    │
│  └──────────────────┴─────────────────────────────────────────┘    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 静态注册广播接收器

```kotlin
// MyReceiver.kt
class MyReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
        when (intent?.action) {
            Intent.ACTION_BOOT_COMPLETED -> {
                Toast.makeText(context, "开机完成", Toast.LENGTH_SHORT).show()
            }
            "com.example.CUSTOM_ACTION" -> {
                val message = intent.getStringExtra("message")
                Toast.makeText(context, "收到: $message", Toast.LENGTH_SHORT).show()
            }
        }
    }
}
```

```xml
<!-- AndroidManifest.xml -->
<receiver
    android:name=".MyReceiver"
    android:exported="true">
    <intent-filter>
        <!-- 监听开机完成 -->
        <action android:name="android.intent.action.BOOT_COMPLETED" />
    </intent-filter>
</receiver>
```

### 3.3 动态注册广播接收器

```kotlin
class MainActivity : AppCompatActivity() {

    private lateinit var receiver: BroadcastReceiver

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // 创建广播接收器
        receiver = object : BroadcastReceiver() {
            override fun onReceive(context: Context?, intent: Intent?) {
                val action = intent?.action
                when (action) {
                    Intent.ACTION_BATTERY_LOW -> {
                        Toast.makeText(context, "电量低！", Toast.LENGTH_SHORT).show()
                    }
                    Intent.ACTION_AIRPLANE_MODE_CHANGED -> {
                        val isOn = intent.getBooleanExtra("state", false)
                        Toast.makeText(context, "飞行模式: $isOn", Toast.LENGTH_SHORT).show()
                    }
                }
            }
        }

        // 注册广播接收器
        val filter = IntentFilter().apply {
            addAction(Intent.ACTION_BATTERY_LOW)
            addAction(Intent.ACTION_AIRPLANE_MODE_CHANGED)
        }
        registerReceiver(receiver, filter)
    }

    override fun onDestroy() {
        super.onDestroy()
        // 取消注册
        unregisterReceiver(receiver)
    }
}
```

### 4.2 读取系统 ContentProvider（通讯录）

```kotlin
class ContactActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_contact)

        // 检查权限
        if (ContextCompat.checkSelfPermission(this, Manifest.permission.READ_CONTACTS)
            != PackageManager.PERMISSION_GRANTED) {
            // 请求权限
            ActivityCompat.requestPermissions(
                this,
                arrayOf(Manifest.permission.READ_CONTACTS),
                100
            )
        } else {
            readContacts()
        }
    }

    override fun onRequestPermissionsResult(
        requestCode: Int,
        permissions: Array<out String>,
        grantResults: IntArray
    ) {
        super.onRequestPermissionsResult(requestCode, permissions, grantResults)
        if (requestCode == 100 && grantResults.isNotEmpty()
            && grantResults[0] == PackageManager.PERMISSION_GRANTED) {
            readContacts()
        }
    }

    private fun readContacts() {
        val contacts = mutableListOf<String>()

        // 查询通讯录
        val cursor = contentResolver.query(
            ContactsContract.CommonDataKinds.Phone.CONTENT_URI,
            null,  // 所有列
            null,  // 没有 WHERE 条件
            null,  // 没有参数
            null   // 默认排序
        )

        cursor?.use {
            while (it.moveToNext()) {
                val nameIndex = it.getColumnIndex(
                    ContactsContract.CommonDataKinds.Phone.DISPLAY_NAME
                )
                val numberIndex = it.getColumnIndex(
                    ContactsContract.CommonDataKinds.Phone.NUMBER
                )

                val name = it.getString(nameIndex)
                val number = it.getString(numberIndex)
                contacts.add("$name: $number")
            }
        }
```

### 4.3 自定义 ContentProvider

```kotlin
// MyContentProvider.kt
class MyContentProvider : ContentProvider() {

    companion object {
        const val AUTHORITY = "com.example.androidlearn.provider"
        const val TABLE_NAME = "users"
        val CONTENT_URI: Uri = Uri.parse("content://$AUTHORITY/$TABLE_NAME")
    }

    private lateinit var dbHelper: UserDatabaseHelper

    override fun onCreate(): Boolean {
        dbHelper = UserDatabaseHelper(context!!)
        return true
    }

    override fun query(
        uri: Uri,
        projection: Array<out String>?,
        selection: String?,
        selectionArgs: Array<out String>?,
        sortOrder: String?
    ): Cursor? {
        val db = dbHelper.readableDatabase
        val cursor = db.query(
            TABLE_NAME,
            projection as? Array<String>,
            selection,
            selectionArgs as? Array<String>,
            null,
            null,
            sortOrder
        )
        cursor.setNotificationUri(context?.contentResolver, uri)
        return cursor
    }

    override fun insert(uri: Uri, values: ContentValues?): Uri? {
        val db = dbHelper.writableDatabase
        val id = db.insert(TABLE_NAME, null, values)
        context?.contentResolver?.notifyChange(uri, null)
        return Uri.withAppendedPath(CONTENT_URI, id.toString())
    }

    override fun update(
        uri: Uri,
        values: ContentValues?,
        selection: String?,
        selectionArgs: Array<out String>?
    ): Int {
        val db = dbHelper.writableDatabase
        val count = db.update(TABLE_NAME, values, selection, selectionArgs as? Array<String>)
        context?.contentResolver?.notifyChange(uri, null)
        return count
    }

    override fun delete(uri: Uri, selection: String?, selectionArgs: Array<out String>?): Int {
        val db = dbHelper.writableDatabase
        val count = db.delete(TABLE_NAME, selection, selectionArgs as? Array<String>)
        context?.contentResolver?.notifyChange(uri, null)
        return count
    }

    override fun getType(uri: Uri): String {
        return "vnd.android.cursor.dir/vnd.$AUTHORITY.$TABLE_NAME"
    }
}
```

```xml
    android:exported="true" />
```

## 七、常见问题

### Q1: Activity 旋转屏幕后数据丢失？
A: 使用 `onSaveInstanceState()` 保存数据，在 `onCreate()` 中恢复。

```kotlin
override fun onSaveInstanceState(outState: Bundle) {
    super.onSaveInstanceState(outState)
    outState.putString("key", "value")
}

override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    val value = savedInstanceState?.getString("key")
}
```

### Q2: Service 在后台被系统杀死怎么办？
A: 使用 `startForeground()` 创建前台服务，或返回 `START_STICKY` 让系统重启服务。

### Q3: Android 8.0+ 广播注册限制？
A: 大部分系统广播必须使用动态注册，不能使用静态注册。自定义广播需要指定包名。
